The Microphysiology Systems (MPS) Database combines the loading, storage, and sharing of experimentally derived assay data and data curated from external reference sources with visualization and analysis tools to provide a powerful platform for designing experimental studies and performing computational modeling.

What's New in the MPS Database

  • Enhanced Options for Overriding the Built-in Grouping/Filtering, Plotting, and Reproducibility Analysis of Assay Data - A left-sidebar offers a wide variety of options for dynamically changing data aggregation grouping and filtering. The sidebar can be toggled on and off to allow flexible use of screen space. An option to manually refresh data aggregation has been provided to allow users to make multiple changes without waiting for the refresh after each change. Filter buttons are now highlighted when active. The sidebars are targeted to the displayed feature (i.e. the options for the reproducibility plots are different than the options for the study summary plots), and new options for customizing individual plots have been added. For more information on the intra-study feature, click here. For more information on the inter-study feature, click here.
  • Integrated Reproducibility Analysis - An intra- and inter-study reproducibility analysis tool is now available in the MPS database. The inter-study tool can be accessed through the Data Analysis icon or tab. This tool offers the user a wide range of options for grouping and filtering data prior to performing graphing and/or reproducibility analysis. The intra-study tool can be accessed through the Study Summary page of the MPS-GUI. For more information on the intra-study feature, click here. For more information on the inter-study feature, click here.
  • MPS Models can be Grouped into a Matrix - MPS Models (i.e. Matrix Items), tested as part of a study, can now be assigned to a Matrix and data can be summarized by Matrix. Matrices can be accessed from the Study Matrices and Items list page of a specific study. For more information on this feature, click here and click to see information on the Built-in Default Grouping/Filtering by Matrix.
  • Integration of Images and Videos - Images and videos can now be associated to a MPS Model (i.e. Matrix Item) within a study. Images and videos are auto-scaled for optimal viewing online and the unaltered image/video is available for download to facilitate performing relative comparisons. Images and videos can be accessed from the Study Matrices and Items list page of a specific study. For more information on this feature, click here.
  • Inter-Study Analysis with Grouping and Filtering - The grouping and filtering feature is a powerful tool that offers the user a wide range of options for grouping and filtering data. The tool can be accessed in several places throughout the database, including: through the Data Analysis icon or tab and the Study Summary page. The tool can be used to customize what data are used (via the filtering feature) and how data will be grouped (via the grouping feature) in subsequent graphing and analysis. For more information on this feature, click here.
  • View Diseases - The View Diseases feature allows users to quickly find information regarding a specific disease. Users can drill down on data stored in the MPS database (study data and data extracted from the clinicaltrials.gov website), and link to clinicaltrials.gov where additional drug trial information can be obtained. The View Diseases feature can be accessed through the MPS Models icon or tab. For more information on this feature, click here.
  • Add Experimental Variables as Settings - This feature allows the Data Provider the ability to enter treatments and/or experimental parameters (such as instrument settings) that impact assay data measurements. These, along with the added compounds, can be used to group and filter data in preparation for visualization and analysis. For more information on adding settings, click here. For more information on using settings in grouping and filtering, click here.

Overview of the MPS Database

  • The Microphysiology Systems (MPS) Database was designed for the management and analysis of MPS data together with curated reference and clinical data. The MPS Database can store curated assay data and associated metadata from a wide variety of plate-based and microfluidic models. Data from external reference sources are brought into the MPS Database to facilitate ease of access to those data and to support integration of data across curated data sources.

    Study Data and Study Components (e.g., MPS Models, Devices, Cell Samples) are manually curated into the MPS Database. Data from some reference data sources are both manually and auto-curated. The MPS Database provides tools to analyze and/or visualize data pulled from external data sources, and also provides tools to analyze curated study data in combination with reference data. Study Data, Study Components, and the Features available in the MPS Database are described briefly in the tables below.

    Many of the terms used in the MPS-GUI and here in the Help are defined in the Glossary. It is recommended that users rely heavily on the Glossary to assist in understanding how to appropriately review, edit, and/or add data to the MPS Database.

  • Role Access to Data and Features
    Registered MPS Users with No Additional Permissions Can view all reference data and associated features. Can view Study component lists (e.g., MPS Models, Devices, Cell Samples). Can view Study Data in Studies that have been designated as open to all registered MPS users, including: viewing Matrix Item setup, study protocols, images, videos, and Study Data.
    Data Group Viewers and Users with Additional Permissions Have all the access of registered MPS Users with No Additional Permissions plus access to Studies to which they have been granted special access. Special access is sometimes granted to organ model devleopers (i.e. acknowledgement access) and/or project funders.
    Data Group Editors Have all the access of Data Group Viewers and Users with Additional Permissions plus the ability to edit their Studies, including: adding/deleting/editing Studies and Study details; adding/deleting/editing Matrices and Matrix Items; uploading assay data; and validating Matrices, Matrix Items, and Studies.
    Data Group Admins Have all the access of Data Group Editors plus the ability to sign off a Study and monitor the acknowledgment status of a Study (e.g., indicating a model developer has reviewed Study data).
    MPS Database Administrators Have all the access of Data Group Admins plus the ability to add/edit/delete new controlled vocabulary (e.g., MPS Models, Devices, Cell Samples) and change user permissions, including establishing users with special access to specific studies.
    Note: Additional permissions can be granted in a multitude of configurations to match individual user needs. For example, any registered user could be granted permission to add/edit Cell Samples, Compounds, MPS Models, etc.

Searching the MPS Database

    • Use a Global Search to search the whole MPS Database from the Home page
      • Go to the MPS Database Home page
      • Type search text in the Global search bar
      • Make preliminary selection (hit enter, click the blue Search button, or mouse down the options list to item of interest and click the left mouse button)
      • Wait for a list of matching information to appear
      • Click to check (or uncheck) boxes in the filter section to further restrict (or unrestrict) matching results (optional)
      • Click on link to be routed to desired location
    • In many places throughout the database, there are lists/tables with search bars located at the top
      • Type a search term in the search bar
      • Type a space and add an additional term(s) (optional)
      • From the list, click on button to View/Edit or click an active link to be routed to desired location

    • Notes
      • Above search bars are options to Copy the table to the clipboard, download the table to CSV, or Print the table
      • The Column visibility button can be used to turn columns on/off in the list/table
      • To sort the list/table, click on the arrow in the column header
      • To sort on multiple columns, click on the arrow on the primary sort column, then hold the shift key and click on addtional column arrows

Instructions for Data Providers

    • Data and metadata collected for inclusion in the MPS Database are categorized based on when and how they are incorporated into the database. These categories are:
      • Study Context information about the MPS Models, assays, compounds, and cells used in the study (these become options in the study component lists)
      • Study Description the study level details (start date, purpose, methods, and description of the results)
      • Study Design the chips level details (when and what cells, treatments, media, and settings were added to each MPS model)
      • Study Data the resulting measured values, images, and videos

    • It should be noted that there is some overlap between these categories, for example, some study design information will likely be included in the model protocol.

    • The Study Description, Study Design, and Study Data can be added directly to the database by a data provider (a registered user who has been granted permission to add study data to the database). Adding new components to the study component lists requires special permission that is not typically granted to data providers.

Controlling User Access

    • A Group is actually a trio of groups. The following is an example for Data Group Rusyn_TAMU
      • Rusyn_TAMU - this is the editor group, sometimes referred to as the group proper
      • Rusyn_TAMU Admin - admins can edit and sign off (or acknowledge for TCD groups) studies
      • Rusyn_TAMU Viewer - can view studies prior to sign off (or acknowledgement for TCD groups)
    • To create and prepare a new Group
      • Go to the MPS Database admin (https://mps.csb.pitt.edu/admin/)
      • Go to the MPS Authentication and Authorization Section
      • Click on Groups and scroll down to review how groups are named
      • Click on the ADD GROUP button on the top right side
      • Give the group a name (for example: my_group)
      • Click the Save and add another button
      • Give the group a name (for example: my_group Admin)
      • Click the Save and add another button
      • Give the group a name (for example: my_group Viewer)
      • Click the SAVE button
      • Return to the admin home page (https://mps.csb.pitt.edu/admin/)
      • Go to the MPS CENTERS AND MODELS Section
      • Scroll to find the Center with which the new group will be associated
      • Click on the center Name to select the center
      • Scroll down to the Groups section and add the new group (the group proper) to the center
      • Scroll down and click the SAVE button

    • Notes
      • It is very easy to forget to attach the group proper to a center. If you forget, you will not be able to use the group.
    • To assign groups to a Study
      • Go to the MPS Database admin
      • Go to the MPS Assays Section
      • Click on Studies
      • Scroll through the groups or use the search bar to find the study
      • Click to open the study
      • Scroll to the Study Data Group and Access Group Info Section (the Data Group should already be selected)
      • Select all Access Groups (add the group proper - not admin or viewer)
      • Scroll to the Assay Study Stakeholders Section
      • Add the stakeholder group(s) (add the group proper - not admin or viewer)

    • Notes
      • Best practice would be to assign appropriate groups at the time of study creation.
      • Data Group Admins and Editors can create a study, but cannot add/edit special permission groups to a study
      • Data Providers must work with MPS Database Admins to set up appropriate groups when/if needed.
      • Clicking on the button to Change Group Access for this Study from the Study Matrices and Items List page will route the user to the Change Study page of the GUI Admin.
    • To assign users to a Group
      • Go to the MPS Database admin
      • Go to the Authentication and Authorization Section
      • Click on Users
      • Scroll through the users or use the search bar
      • Click to open the user details
      • Change group membership as needed
      • Save the changes

    • Notes
      • Users should be assigned to appropriate groups when they register
      • Users may have different roles in different groups

      • Rusyn_TAMU - this is the editor group, sometimes referred to as the group proper

      • Rusyn_TAMU Viewer - can view studies prior to sign off

      • TCD_BBB_Vandy Viewer - members are users who can see a study but cannot acknowledge it

      • NCATS - group must be present, but should not have members

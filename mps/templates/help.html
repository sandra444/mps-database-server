    Searching the MPS-Db

      • Use a Global Search to search the whole MPS-Db from the Home page
        • Go to the MPS-Db Home page
        • Type search text in the Global search bar
        • Make preliminary selection (hit enter, click the blue Search button, or mouse down the options list to item of interest and click the left mouse button)
        • Wait for a list of matching information to appear
        • Click to check (or uncheck) boxes in the filter section to further restrict (or unrestrict) matching results (optional)
        • Click on link to be routed to desired location
      • In many places throughout the database, there are lists/tables with search bars located at the top
        • Type a search term in the search bar
        • Type a space and add an additional term(s) (optional)
        • From the list, click on button to View/Edit or click an active link to be routed to desired location

      • Notes
        • Above search bars are options to Copy the table to the clipboard, download the table to CSV, or Print the table
        • The Column visibility button can be used to turn columns on/off in the list/table
        • To sort the list/table, click on the arrow in the column header
        • To sort on multiple columns, click on the arrow on the primary sort column, then hold the shift key and click on additional column arrows

    Customizing Queries, Plots, and Analyses (Overriding the Defaults)

      • Customized grouping and filtering is available in the MPS-Db via a "Sidebar" where data are available for review
        • If hidden, click the Toggle Sidebar button to show the sidebar
        • Click on the parameter menus (Study Parameters, Cell Parameters, Compound Parameters, Setting Parameters) to see options
        • To filter - click on the filter symbol
          • To select data, check the box next to the data
          • To omit data, uncheck the box next to the data
        • To group - check/uncheck the box in the Group column
          • Check the box to include parameter in grouping
          • Uncheck the box to remove parameter from grouping
        • REPEAT for all attributes to override
      • Additional graphing options are available on an individual plot
        • Right click on an individual plot
        • Check the desired options
        • Click Apply to Plot

      • Notes

    Instructions for Data Providers

      • Data and metadata collected for inclusion in the MPS-Db are categorized based on when and how they are incorporated into the database. These categories are:
        • Study Context – information about the MPS Models, assays, compounds, and cells used in the study (these become options in the study component lists)
        • Study Description – the study level details (start date, purpose, methods, and description of the results)
        • Study Design – the chips level details (when and what cells, treatments, media, and settings were added to each MPS model)

    Controlling User Access

      • There are now a lot of options for controlling and sharing data in the MPS-Db.
      • All of the options work on a "study" (not on individual data points).
      • Access is controlled by assigning permissions to users and by assigning groups to studies.
      • Looking first at setting permissions to a study, in the admin, in the MPS Assays section, select Studies. Notice in the list the fields: Signed Off By, Stakeholder Display, Access Group Display, Collaboration Group Display, Restricted, Keep Private Indefinitely (Locked). These summarize if and how each study is being shared.

      • Examining each field:
        • Signed Off By - If this has a user name in it, it means that the study is following the sign off, acknowledgement, embargo process. A sign off does not open the study to the public, even if there is no stakeholder. If there is no stakeholder, the study is open to Access Groups.
        • Stakeholder Display - If there is something here, it means there is a stakeholder for the study.
        • Access Group Display - If there is something here, users in these groups can see the study after sign off (if there are not stakeholders) or after stakeholder acknowledgement.
        • Collaboration Group Display - This is a special feature that allows sharing a study, at any time, to users in these groups.
        • Restricted - Until this is unchecked, a study is closed to the public. It can be unchecked by an admin at any time. Studies in the embargo period (after sign off and acknowledgement) will be unchecked one year and two weeks after sign off (automated process), UNLESS the study is 'locked'.
        • Keep Private Indefinitely (Locked) - Locking a study stops the study from being released to the next level (blocks automatic migration to public access - public if acknowledged, access if stakeholders, stakeholders if signed off).

      • Notes
        • What happens if the 'Use in Calculations' box is check? The study becomes available in the Compound Report.
        • Collaborator groups are completely controlled by the admins (no front-end GUI).
        • To open a study to the public immediately (i.e. bypass sign off, acknowledgment, embargo) must by done by the admin (uncheck the restricted box), but data providers can check the 'Use in Calculations' box.
        • While the use is sometimes indicated in the group name, there is no real difference between groups (data, stakeholder, access, collaborator). Use caution. Even though a data group and a stakeholder group is associated with the same institution, the users in the groups maybe different.

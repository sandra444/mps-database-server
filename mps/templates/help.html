What's New in the MPS Database

  • Integrated Reproducibility Analysis - An intra- and inter-study reproducibility analysis tool is now available in the MPS database. The inter-study tool can be accessed through the Data Analysis tab or icon. This tool offers the user a wide range of options for grouping and filtering data prior to performing graphing or reproducibility analysis. The intra-study tool can be accessed through the Study Summary page. For more information on this feature, click here.
  • Matrix Items in a Matrix - Matrix Items can now be assigned to a Matrix and data can be summarized by Matrix. Matricies can be accessed from the Study Matrices and Items list page of a specific study. For more information on this feature, click here.
  • Integration of Images and Videos - Images and videos can now be associated to a Matrix Item within a study. Images and videos are auto-scaled for optimal viewing online and the unaltered image/video is available for downloaded to facilitate performing relative comparisons. Images and videos can be accessed from the Study Matrices and Items list page of a specific study. For more information on this feature, click here.
  • Inter-Study Analysis with Grouping and Filtering - The grouping and filtering feature is a powerful tool that offers the user a wide range of options for grouping and filtering data. The tool can be accessed in several places throughout the database, including: through the Data Analysis icon or tab and the Study Summary page. The tool can be used to customize what data are used (via the filtering feature) and how data will be grouped (via the grouping feature) in subsequent graphing and analysis. For more information on this feature, click here.
  • Diseases Portal - The Diseases portal allows users to quickly find information regarding a specific disease. The protal lets the user drill down on data stored in the MPS database (study data and data extracted from the clinicaltrials.gov), and provides users with a links to clinicaltrials.gov where additional drug trial information can be obtained. The portal can be accessed through the Models icon or tab. For more information on this feature, click here.
  • Non-Compound Treatments and/or Settings - This feature allows the Data Provider the ability to enter treatments and/or experimental parameters (such as instrument settings) that impact assay data measurements. These, along with the added compounds, can be used to group and filter data on preparation for visualization and analysis. For more information on this feature, click here.

Searching the MPS Database

    • Use a Global Search to search the whole MPS Database from the Home page
      • Go to the MPS Database Home page
      • Type search text in the Global search bar
      • Make preliminary selection (hit enter, click the blue Search button, or mouse down the options list and click the left mouse button)
      • Wait for a list of matching information to appear
      • Click to check boxes in the filter section to further limit matching results (optional)
      • Click on link to be routed to desired location

    • Notes:
      • There are search bars in various locations in the MPS-GUI
      • Most search bars limit results on entry (no Enter or Search button required)
      • In all search bars, include a space between search terms to conduct a multi-term search

Overview of the MPS Database

  • The Microphysiology Systems (MPS) Database was designed for the management and analysis of MPS data together with curated reference and clinical data. The MPS Database can store curated assay data and associated metadata from a wide variety of plate-based and microfluidic models. Data from external reference sources are brought into the MPS Database to facilitate ease of access to those data and to support integration of data across curated data sources.

    All Study Data and Study Components (e.g., Models, Devices, Cell Samples) are manually curated into the MPS Database. Data from some references data sources are both manually and auto-curated. The MPS Database provides tools to analyze and/or visualize data pulled from external data sources, and also provides tools to analyze curated study data combined with reference data. Study Data, Study Components, and the Features of available in the MPS Database are describing briefly in the tables below.

    Many of the terms used in the MPS-GUI and here in the Help are defined in the Glossary. It is recommened that you rely heavily on the Glossary to assist in understanding how to appropriately review, edit, and/or add data to the MPS Database.

  • Role Access to Data and Features
    Registered MPS Users with No Additional Permissions Can view all reference data and associated features. Can view Study component lists (e.g., Models, Devices, Cell Samples). Can view Study Data in Studies that have been designated as open to all registered MPS users, including: viewing Matrix Item setup, study protocols, images, videos, and Study Data.
    Data Group Viewers and Users with Additional Permissions Have all the access of registered MPS Users with No Additional Permissions plus access to Studies to which they have been granted special access.
    Data Group Editors Have all the access of Data Group Viewers and Users with Additional Permissions plus the ability to edit their Studies, including: adding/deleting/editing Studies and Study details; adding/deleting/editing Matricies and Matrix Items; uploading assay data; and validating Matricies, Matrix Items, and Studies.
    Data Group Admins Have all the access of Data Group Editors plus the ability to sign off a Study and monitor the acknowledgment status of a Study (e.g., indicating a model developer has reviewed their Study data).
    MPS Database Administrators Have all the access of Data Group Admins plus the ability to add/edit/delete new controlled vocabulary to lists (e.g., Models, Devices, Cell Samples). Can change user permissions, including establishing users with acknowledgment access to specific studies.

Adding Assay Data Step 1: Adding/Editting a Study

    • To Add/Edit a Study:
      • Check to make sure that all needed components are available in the database Study Components
      • Click on the Add Study button or the View/Edit button next to the Study of interest to be routed to the Study Matrices and Items list page
      • Click on the Edit This Study button and add/edit the required information
        • Group select a Data Group
        • Study Type indicate the general purpose of the Study (one is required; check all that apply)
        • Study Name use an informative name (e.g., "Kidney-Liver Vitamin C Metabolism" is more informative than "Study-1")
        • Start Date - start of experiment (Day 0)
        • Description - informative description
        • Use in Calculations - checking Use in Calculations tells the database to use this study's data in predictive modeling and other calculations
        • Protocol File a Word or pdf file containing the Study protocol
        • Image to Clarify A Study recommend a picture of the setup or an illustration of the experimental protocol
        • For Integrated Chip Studies select configuration if performing an integrated Study
        • Assays for each combination of Method-Target-Unit of assay data that will be loaded to the study, add an assay row to the section
        • Supporting Data attach files that will assist other database users in understanding the Study design and/or assay data
        • Click the Submit button to save changes
      • For information on continuing to add MPS models and assay data to the Study (Step 2), click here

    • Tips:
      • If new components are needed, contact an MPS Database administrator
      • Everything in the Study can be edited at any time until the Study Data Group Admin signs off the Study (then it is locked)
      • Data contained in files attached as Supporting Data are NOT available for visualization, analysis, or used in calculations within the MPS-GUI
      • All Combinations of Method-Target-Unit must be listed in the Assays Section of the Study Details page prior to uploading assay data
    • A Study, each Matrix, and/or each Matrix Item can be flagged and annotated
      • Clicking places the flag
      • Clicking Notes brings up an input dialog where annotations can be added

    • Tips:
      • Flags and notes are fully reversible until a Study is signed off
      • The annotation process works the same way for a Study, a Matrix, and a Matrix Item
    • When performing Integrated Studies:
      • When setting up a Study, Models can be associated
      • Each Model is assigned a Label, which can be one or two characters long
      • Sequence Number shows the order of input for the Models
      • Output lists all of the other Models the current Models feeds into
      • Output should be a list of labels separated by dashes (e.g. "AB-CD" means the model is connected to AB and CD)
      • Coupling indicates whether or not the model is directly connected to the other models
      • Click Add Model to add another row
    • To view Matrix details, from the Study Matrices and Items list page:
      • Use the search bar or scroll through the list to find the Matrix of interest
      • Review information provided
      • Click on the View/Edit button next to the Matrix of interest
      • Click on the Show Settings button to see non-compound treatments/conditions/settings
      • Click on the Show Compounds to see compound treatments
      • Click on the Show Cells button to see cells added toMatrix Items
      • Review the setup of all theMatrix Items as a group
      • The Grouping/Filtering section enables changing what/how data are displayed (for more information on how to use this feature, click here)
      • The Current Data section presents visualizations for assay data associated with the Matrix Items in this Matrix (for more information on how to use this feature, click here)

    • Notes:
      • To review how to navigate to the Study Matrices and Items list page, click here
      • The Current Data sections of the Study Summary page, the Matrix Setup and Data page, and the Matrix Item Setup and Data page all work the same way

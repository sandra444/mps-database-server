What's New in the MPS Database

  • Integrated Reproducibility Analysis - An intra- and inter-study reproducibility analysis tool is now available in the MPS database. The inter-study tool can be accessed through the Data Analysis icon or tab. This tool offers the user a wide range of options for grouping and filtering data prior to performing graphing and/or reproducibility analysis. The intra-study tool can be accessed through the Study Summary page of the MPS-GUI. For more information on this feature, click here.
  • Matrix Items in a Matrix - Matrix Items can now be assigned to a Matrix and data can be summarized by Matrix. Matrices can be accessed from the Study Matrices and Items list page of a specific study. For more information on this feature, click here.
  • Integration of Images and Videos - Images and videos can now be associated to a Matrix Item within a study. Images and videos are auto-scaled for optimal viewing online and the unaltered image/video is available for download to facilitate performing relative comparisons. Images and videos can be accessed from the Study Matrices and Items list page of a specific study. For more information on this feature, click here.
  • Inter-Study Analysis with Grouping and Filtering - The grouping and filtering feature is a powerful tool that offers the user a wide range of options for grouping and filtering data. The tool can be accessed in several places throughout the database, including: through the Data Analysis icon or tab and the Study Summary page. The tool can be used to customize what data are used (via the filtering feature) and how data will be grouped (via the grouping feature) in subsequent graphing and analysis. For more information on this feature, click here.
  • Diseases Portal - The Diseases portal allows users to quickly find information regarding a specific disease. The portal lets the user drill down on data stored in the MPS database (study data and data extracted from the clinicaltrials.gov website), and provides users with a links to clinicaltrials.gov where additional drug trial information can be obtained. The portal can be accessed through the Models icon or tab. For more information on this feature, click here.
  • Non-Compound Treatments and/or Settings - This feature allows the Data Provider the ability to enter treatments and/or experimental parameters (such as instrument settings) that impact assay data measurements. These, along with the added compounds, can be used to group and filter data in preparation for visualization and analysis. For more information on this feature, click here.

Overview of the MPS Database

  • The Microphysiology Systems (MPS) Database was designed for the management and analysis of MPS data together with curated reference and clinical data. The MPS Database can store curated assay data and associated metadata from a wide variety of plate-based and microfluidic models. Data from external reference sources are brought into the MPS Database to facilitate ease of access to those data and to support integration of data across curated data sources.

    Study Data and Study Components (e.g., Models, Devices, Cell Samples) are manually curated into the MPS Database. Data from some references data sources are both manually and auto-curated. The MPS Database provides tools to analyze and/or visualize data pulled from external data sources, and also provides tools to analyze curated study data in combination with reference data. Study Data, Study Components, and the Features available in the MPS Database are describing briefly in the tables below.

    Many of the terms used in the MPS-GUI and here in the Help are defined in the Glossary. It is recommended that users rely heavily on the Glossary to assist in understanding how to appropriately review, edit, and/or add data to the MPS Database.

  • Role Access to Data and Features
    Registered MPS Users with No Additional Permissions Can view all reference data and associated features. Can view Study component lists (e.g., Models, Devices, Cell Samples). Can view Study Data in Studies that have been designated as open to all registered MPS users, including: viewing Matrix Item setup, study protocols, images, videos, and Study Data.
    Data Group Viewers and Users with Additional Permissions Have all the access of registered MPS Users with No Additional Permissions plus access to Studies to which they have been granted special access. Special access is sometimes granted to organ model devleopers (i.e. acknowledgement access) and/or project funders.
    Data Group Editors Have all the access of Data Group Viewers and Users with Additional Permissions plus the ability to edit their Studies, including: adding/deleting/editing Studies and Study details; adding/deleting/editing Matrices and Matrix Items; uploading assay data; and validating Matrices, Matrix Items, and Studies.
    Data Group Admins Have all the access of Data Group Editors plus the ability to sign off a Study and monitor the acknowledgment status of a Study (e.g., indicating a model developer has reviewed Study data).
    MPS Database Administrators Have all the access of Data Group Admins plus the ability to add/edit/delete new controlled vocabulary (e.g., Models, Devices, Cell Samples) and change user permissions, including establishing users with special access to specific studies.

Searching the MPS Database

    • Use a Global Search to search the whole MPS Database from the Home page
      • Go to the MPS Database Home page
      • Type search text in the Global search bar
      • Make preliminary selection (hit enter, click the blue Search button, or mouse down the options list and click the left mouse button)
      • Wait for a list of matching information to appear
      • Click to check (or uncheck) boxes in the filter section to further restrict (or unrestrict) matching results (optional)
      • Click on link to be routed to desired location
    • In many places throughout the database, there are lists/tables with search bars located at the top
      • Type a search term in the search bar
      • Type a space and add an additional term(s)(optional)
      • From the list, click on button to View/Edit or click an active link to be routed to desired location

    • Notes
      • Above search bars are options to Copy the table to the clipboard, download the table to CSV, or Print the table
      • The Column visibility button can be used to turn columns on/off in the list/table
      • To sort the list/table, click on the arrow in the column header
      • To sort on multiple columns, click on the arrow on the primary sort column, then hold the shift key and click on addtional column arrows

Instructions for Data Providers

    • To Add/Edit a Study
      • Check to make sure that all needed components are available in the database Study Components
        • To have a new components added, contact an MPS Database Administrator
      • Click on the Add Study button or the View/Edit button next to the Study of interest to be routed to the Study Matrices and Items list page
      • Click on the Edit This Study button and add/edit the required information
        • Group – select a Data Group (the team performing the study)
        • Study Type – indicate the general purpose of the Study (one is required; check all that apply)
        • Study Name – use an informative name (e.g., "Kidney-Liver Vitamin C Metabolism" is more informative than "Study-1")
        • Start Date - start of experiment (Day 0)
        • Description - informative description
        • Use in Calculations - checking Use in Calculations tells the database to use this study's data in predictive modeling and other calculations
        • Protocol File – a Word or pdf file containing the Study protocol
        • Image to Clarify A Study – recommend a picture of the setup or an illustration of the experimental protocol
        • For Integrated Chip Studies – select configuration if performing an integrated Study
        • Assays – for each combination of assay Method-Target-Unit used, add a row
        • Supporting Data – attach files that will assist other database users in understanding the Study design and/or assay data (other than the protocol file, which is added separately)
      • Click the Submit button to save changes
      • For information on continuing to add MPS models and assay data to the Study (Step 2), click here

    • Notes
      • Everything in the Study can be edited at any time until the Study Data Group Admin signs off the Study (then it is locked)
      • Data contained in files attached as Supporting Data are NOT available for visualization, analysis, or used in calculations within the MPS-GUI
      • All combinations of Method-Target-Unit must be listed in the Assays Section of the Study Level Details page prior to uploading assay data
    • A Study, each Matrix, and/or each Matrix Item can be flagged and annotated
      • Clicking places the flag
      • Clicking Notes brings up an input dialog where annotations can be added

    • Notes
      • Flags and notes are fully reversible until a Study is signed off
      • The annotation process works the same way for a Study, a Matrix, and a Matrix Item
      • A Study is annotated at the bottom of the Study Level Details page
      • A Matrix is annotated at the bottom of the Matrix Setup and Data page
      • A Matrix Item is annotated at the bottom of the Item Setup and Data page
    • A Study, each Matrix, and/or each Matrix Item can be validated
      • Study sign off is done by the Data Group Admin by clicking the View/Edit Sigh Off Status of this Study button on the Study Matrices and Items list page
      • A Matrix is validated from the Matrix Setup and Data page
      • A Matrix Item is validated from the Item Setup and Data page
    • When performing Integrated Studies:
      • When setting up a Study, Models can be associated
      • Each Model is assigned a Label, which can be one or two characters long
      • Sequence Number shows the order of input for the Models
      • Output lists all of the other Models the current Models feeds into
      • Output should be a list of labels separated by dashes (e.g. "AB-CD" means the model is connected to AB and CD)
      • Coupling indicates whether or not the model is directly connected to the other models
      • Click Add Model to add another row

Controlling User Access

    • To assign groups to a Study
      • Go to the MPS Database admin
      • Go to the MPS Assays Section
      • Click on Studies
      • Scroll through the groups or use the search bar to find the study
      • Click to open the study
      • Scroll to the Study Data Group and Access Group Info Section (the Data Group should already be selected)
      • Select all Access Groups (add the group proper - not admin or viewer)
      • Scroll to the Assay Study Stakeholders Section
      • Add the stakeholder group(s) (add the group proper - not admin or viewer)

    • Notes
      • Best practice would be to assign appropriate groups at the time of study creation.
      • Data Group Admins and Editors can create a study, but cannot add/edit special permission groups to a study
      • Data Providers must work with MPS Database Admins to set up appropriate groups when/if needed.
      • Clicking on the button to Change Group Access for this Study from the Study Matrices and Items List page will route the user to the Change Study page of the GUI Admin.
    • To assign users to a Group
      • Go to the MPS Database admin
      • Go to the Authentication and Authorization Section
      • Click on Users
      • Scroll through the users or use the search bar
      • Click to open the user details
      • Change group membership as needed
      • Save the changes

    • Notes
      • Users should be assigned to appropriate groups when they register
      • Users may have different roles in different groups

    • The term Data Group actually refers to three groups, one group for each role. The following is an example for Data Group Rusyn_TAMU
      • Rusyn_TAMU - this is the editor group, sometimes referred to as the group proper
      • Rusyn_TAMU Admin - admins can edit and sign off studies
      • Rusyn_TAMU Viewer - can view studies prior to sign off

    • The term Stakeholder Group also refers to three groups, one group for each role. The following is an example for TCD_BBB_Vandy
      • TCD_BBB_Vandy - this group must be present, but should not have members
      • TCD_BBB_Vandy Admin – members are users who can acknowledge a study
      • TCD_BBB_Vandy Viewer - members are users who can see a study but cannot acknowledge it

    • The term Access Group also refers to three groups, one group for each role. The following is an example for the TCD_BBB_Vandy
      • NCATS - group must be present, but should not have members
      • NCATS Admin - group should not have members
      • NCATS Viewer - members are users with special access
